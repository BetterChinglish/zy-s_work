为什么需要定位

标准流，浮动都无法完成的效果，需要定位
1--浮动可以让多个块级盒子没有间隙的横向排列
2--定位可以让盒子自由移动在某个盒子内，或者固定在屏幕中某个位置，并且可以压住其他盒子

定位组成

定位=定位模式+边偏移
定位模式：指定一个元素在文档中的定位方式。
边偏移：决定该元素的最终位置。

定位模式（4个）

通过position属性来设置
1--静态定位 static
2--相对定位 relative
3--绝对定位 absolute
4--固定定位 fixed

边偏移（4个）

top，bottom，right，left
示例：top：80px
顶端偏移量，定义元素相对于父元素上边线的距离

静态定位（默认定位,无定位，按标准流）

语法：
选择器 { position：static； }

相对定位

语法：
选择器 { position：relative top\bottom\right\left:px; }
移动位置时，相对于自原来的位置（自恋型）

绝对定位

语法：
选择器 { position：absolute； }
移动位置时，相较于祖先元素的位置（拼爹型）
特点：
1--如果没有祖先元素或者祖先元素没有定位，则以浏览器为准定位。
2--如果祖先元素有定位，则以最近一级的有定位的祖先元素为参考点移动位置
3--绝对定位不在占有原先的位置（脱标），位置将被别的盒子占领。

子绝父相的由来（绝对定位和相对定位的使用场景)

子级使用绝对定位，父级使用相对定位
1--绝对定位不占用位置，会导致后来设置的盒子破坏结构，所以父级盒子最好用相对定位
2--相对定位则相反，适合子级盒子

固定定位

将元素固定于浏览器可视区的位置。
主要使用场景：
浏览器页面滚动时，元素位置不改变
语法：
选择器 { position：fixed； }
特点：
以浏览器可视窗口为参照点移动元素

固定定位贴到版心

小算法：
先让盒子以浏览器可视区域left：50%
再让盒子以版心宽度margin-left：xx；

粘性定位（了解

可认为是相对定位和固定定位的混合
语法：
选择器 { position：sticky； }
特点：
1--以可视窗口为参照物
2--占有原先位置
3--必须有方位之一才有效果
4--兼容性较差，ie不支持

定位叠放次序

使用z-index来控制盒子的前后顺序(Z轴)
语法：
选择器 { z-index：1； }
数值可以是正整数，负整数或0，数值越大，贴脸越近。

定位的拓展

1·绝对定位的盒子居中

绝对盒子不可以通过margin：0 auto水平居中
和前面的小算法一样
1，left走50%
2，margin负值，往左走自己的一半

2·定位的特殊性

绝对定位和固定定位和浮动有点相似
1，行内元素添加绝对或固定定位，可以设置宽高
2，块级元素添加绝对或固定定位，如果不给宽高，则默认大小是内容大小

3·脱标的盒子不会触发外边距塌陷
4·绝对定位（固定定位）会完全压住盒子

浮动元素只会压住他下面的标准流盒子，但不会压住下面标准流盒子里面的文字和图片
绝对定位和固定定位会压住下面标准流的所有内容
浮动不会压住文字，是因为浮动产生的目的最初是为了做文字环绕效果的
